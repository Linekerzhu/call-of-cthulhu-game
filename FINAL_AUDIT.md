## ⚠️ 关键风险（可能导致项目失败）

### 风险 #1：14天工期过于乐观 🔴 HIGH

**分析：**
```
plan_roadmap.md 规划：
- 14天 × 每天9.5小时 = 133小时
- 50个功能点
- 平均每功能点 2.66小时

实际情况：
- 调试和Bug修复通常占30-40%时间
- 设计细节在开发中会不断调整
- 第7天里程碑（4徽章可用）实际上需要4-5天

乐观估计需要：18-21天
保守估计需要：25-30天
```

**影响：**
-  deadline 压力下可能牺牲质量
-  核心功能可能无法全部完成
-  测试不充分导致发布后Bug多

**缓解方案：**
```
方案A：延长到21天（推荐）
- 每天工作8小时，周末休息
- 3周完成，质量有保障

方案B：严格MVP，再砍功能
- 只保留2个徽章（暴躁鸭+铁壳龟）
- 删除泡泡鱼和跳跳兔（v1.1添加）
- 删除事件系统（只有战斗和商店）
- 14天可能完成

方案C：分阶段发布
- Week 1：核心战斗（能打牌战斗）
- Week 2：地图流程（能通关）
- 边发布边更新
```

---

### 风险 #2：徽章系统复杂度被低估 🔴 HIGH

**分析：**
```
plan_content.md 中每个徽章有：
- 1个核心机制（怒气/守护/元素/灵动）
- 6张专属卡（其中3张是非专属的"通用卡"）
- 复杂的联动效果

实际开发工作量：
- 怒气系统：需要跟踪怒气值、计算伤害加成、UI显示 → 4-6小时
- 守护系统：需要检测相邻、伤害转移逻辑 → 6-8小时
- 元素系统：需要元素计数器、自动释放逻辑 → 6-8小时
- 灵动系统：需要移动计数、抽卡触发 → 4-6小时
- 专属卡效果：24张卡 × 30分钟 = 12小时
- 徽章切换逻辑：数据清理、卡池管理 → 4-6小时

总计：约40-50小时（5-6天），不是2天
```

**影响：**
- 徽章系统无法按计划完成
- 核心玩法受损

**缓解方案：**
```
建议采用方案B：
- MVP只保留2个徽章
- 暴躁鸭（怒气机制相对简单）
- 铁壳龟（守护机制直观）
- 泡泡鱼和跳跳兔移到v1.1

这样徽章系统工作量减半
```

### 风险 #3：AI系统未充分考虑 🟡 MEDIUM

**分析：**
```
plan_rules_mvp.md 中AI描述：
- "敌人会向玩家移动"
- "根据距离决定攻击或移动"

实际需要的AI：
- 寻路算法（BFS/A*）
- 意图决策树（什么时候攻击/防御/蓄力）
- 多敌人协调（避免挤在一起）
- Boss多阶段切换

roadmap分配：Day 3-4（2天）
实际可能需要：3-4天
```

**影响：**
- AI行为怪异，体验差
- 敌人卡住或做出愚蠢决策

**缓解方案：**
```
- 使用简单的规则式AI（if-else）而非复杂算法
- 预定义行为模式（如"攻击→攻击→防御"循环）
- 预留1天专门调试AI
```

### 风险 #5：内容量过大 🟡 MEDIUM

**分析：**
```
plan_content.md 内容：
- 30张卡（需要平衡测试）
- 11种敌人（不同的AI和数值）
- 3个Boss（多阶段机制）
- 3层地图（不同的节点配置）

平衡测试需要：
- 每种徽章通关3次 = 12局
- 每局15分钟 = 3小时纯游戏时间
- 调整和Bug修复 = 2-3倍时间
- 总计：10-15小时
```

**缓解方案：**
```
- MVP只保证"能通关"，不保证"平衡"
- 发布后根据玩家反馈调整
- 设置"简单模式"（敌人HP-30%）保底
```

## 📋 文档一致性问题

### 问题 #1：卡牌数量不一致

```
plan_rules_mvp.md:  "MVP 30张卡"
plan_content.md:   实际设计36张卡
                    （基础6 + 暴躁鸭6 + 铁壳龟6 + 泡泡鱼6 + 跳跳兔6 = 30张徽章专属
                     但还有基础卡6张，总共36张）

问题：基础卡和徽章专属卡有重叠吗？
      例如【大嘴巴子】是独立的还是算在暴躁鸭里？
```

**建议：**
```
明确数量：
- 基础卡：6张（所有角色都有）
- 徽章专属卡：每个徽章6张（其中3张是"通用"的，3张是真正专属的）
- 总计：6 + 24 = 30张

需要修正plan_content.md中的表格
```

### 问题 #2：地形数量不一致

```
plan_rules_mvp.md: "3种地形：平地、森林、高地"
plan_ui.md:        "5种地形：平地、森林、高地、陷阱、障碍"

问题：MVP到底几种地形？
```

**建议：**
```
严格按MVP文档：3种地形
- 平地（默认）
- 森林（+防御）
- 高地（+伤害）

陷阱和障碍移到v2.0
```

### 问题 #3：连携系统描述不一致

```
plan_rules_mvp.md: "只有相邻+30%伤害，删除三角包围"
plan_content.md:   "纸片人连携"和"守护替承伤"描述混在一起

问题：铁壳龟的"守护"和通用"连携"是同一个机制吗？
```

**建议：**
```
明确区分：
- 通用连携：玩家+纸片人相邻 → 伤害+30%（MVP保留）
- 铁壳龟守护：替纸片人承伤（铁壳龟专属）

这是两个独立的机制
```
