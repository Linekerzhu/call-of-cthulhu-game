# 🗺️ 深渊召唤 MVP开发路线图

> **目标**：2周内完成可玩的MVP版本  
> **人力**：1人全职开发  
> **技术栈**：原生JavaScript + HTML5 + CSS3 + PixiJS（可选）  
> **主题**：克苏鲁神话 - 疯狂、深渊、古神

---

## 📊 项目总览

### 功能清单（MVP）

| 模块 | 功能点 | 优先级 | 预计工时 |
|------|--------|--------|----------|
| **核心框架** | 游戏循环、状态管理 | P0 | 2天 |
| **战棋系统** | 网格、移动、地形 | P0 | 2天 |
| **战斗系统** | 意图、伤害计算 | P0 | 2天 |
| **卡牌系统** | 手牌、打出、效果 | P0 | 2天 |
| **徽章系统** | 3个徽章、专属卡 | P1 | 1.5天 |
| **地图系统** | 节点、流程 | P1 | 1天 |
| **UI系统** | 所有界面 | P1 | 2天 |
| **音效音乐** | 基础音效 | P2 | 0.5天 |
| **测试优化** | Bug修复、性能 | P2 | 1天 |
| **合计** | | | **14天** |

## 🎯 Phase 1：核心框架（第1-2天）

### 目标
搭建可运行的基础框架，能显示网格和玩家角色。

### 任务清单

#### Day 1：项目搭建 + 基础渲染
- [ ] 创建项目结构（index.html + css + js）
- [ ] 实现响应式布局框架
- [ ] 集成PixiJS（或使用DOM+CSS）
- [ ] 实现基础网格渲染（5×8）
- [ ] 玩家角色显示和移动（点击格子）

**验收标准**：
```
✅ 打开网页能看到网格
✅ 点击格子玩家能移动过去
✅ 不同屏幕尺寸自适应
✅ 代码结构清晰，有注释
```

#### Day 2：状态管理 + 游戏循环
- [ ] 实现GameState对象
- [ ] 实现游戏循环（update/render分离）
- [ ] 实现回合制切换（玩家回合/敌人回合）
- [ ] 基础输入处理系统
- [ ] 简单的战斗日志UI

**验收标准**：
```
✅ 能切换回合（有UI提示"你的回合"/"敌人回合"）
✅ 状态改变能正确渲染
✅ 代码支持扩展（添加新功能不重构）
```


## 🆘 风险预案

| 风险 | 可能性 | 应对方案 |
|------|--------|---------|
| 某功能超时 | 高 | 删减功能（如先不做音效） |
| 性能不达标 | 中 | 改用DOM替代PixiJS |
| 玩法不好玩 | 中 | 第7天内部测试，及时调整 |
| 手机适配难 | 中 | 先保证手机竖屏，桌面延后 |
