## 🔴 严重问题（必须解决）

### 问题 #1：熟练度系统与15分钟节奏矛盾

**描述：**
- 熟练度从E到S需要打出15张同类卡
- 卡组上限25张，且是混合卡组（攻击+防御+移动）
- 单局只有15分钟，大概15-20场战斗
- 每场战斗大概打出5-8张卡

**计算：**
```
假设玩家专注某一类卡（如攻击卡）
攻击卡大概占卡组 40% = 10张
每场战斗循环卡组 2-3次
每场打出攻击卡约 6-10张
15场战斗 × 6张 = 90张攻击卡
看起来够，但问题在于：
- 玩家可能在第5场才拿到想要的攻击卡
- 可能第10场才形成稳定循环
- 实际能达到S级的局数很少
```

**影响：**
- 大部分玩家体验不到S级和终极卡
- 熟练度系统形同虚设

**解决方案：**
```
方案A：降低门槛
- E→D：1张
- D→C：3张（累计4）
- C→B：5张（累计9）
- B→A：5张（累计14）
- A→S：6张（累计20）

方案B：改变计算方式
- 不是"打出多少张"，而是"造成多少伤害/获得多少格挡"
- 例如：累计造成100点剑伤害 = S级

方案C：跨局累计（改变核心设计）
- 熟练度不清零，但加成也 weaker
- 变成长期养成

推荐：方案A + 明显的升级提示特效
```

### 问题 #2：连携系统的三角包围无法实现

**描述：**
- 玩家是单人角色
- 三角包围需要3个单位围住敌人
- 文档中只有1张召唤卡【集结号角】
- 而且幻象只持续1回合

**影响：**
- 三角包围机制几乎无法触发
- 连携系统设计浪费

**解决方案：**
```
方案A：增加召唤物数量
- 每个流派都有至少2张召唤卡
- 召唤物可以持续存在直到被击败

方案B：改变连携定义
- 不需要"单位"，只需要"占据格子"
- 玩家可以放置"旗帜""陷阱"等占据格子
- 3个非敌方子形成包围

方案C：简化连携
- 删除三角包围，只保留相邻连携
- 相邻连携只需要玩家+1个召唤物/幻象

推荐：方案C（简化，15分钟游戏不适合太复杂）
```

### 问题 #3：资源系统过于复杂（5种资源）

**描述：**
1. **移动力** - 每回合2点，用于移动
2. **能量** - 卡牌费用
3. **怒气**（狂战士）- 受伤积累，加伤害
4. **正义**（圣骑士）- 守护触发，换免费防御
5. **灵动**（游侠）- 移动抽卡

**影响：**
- 新玩家学习成本极高
- 15分钟内要理解5种资源+卡牌+地形+连携
- 认知过载

**解决方案：**
```
方案A：统一资源
- 删除"移动力"，移动也用"能量"
- 例如：移动1格 = 1能量
- 这样只有能量+徽章特殊资源

方案B：简化徽章资源
- 怒气、正义、灵动都变成"能量"的变种
- 例如：怒气 = 受到伤害获得临时能量

方案C：渐进解锁
- 第1层只有基础（能量+移动力）
- 获得徽章后才解锁对应资源
- 教学引导

推荐：方案A（最简洁）
```

### 问题 #4：高低差系统的技术实现难度

**描述：**
- 当前代码使用曼哈顿距离（2D）
- 高低差需要Z轴坐标
- 射程计算需要考虑Z轴：
  - 高地打低地：射程+1
  - 低地打高地：射程-1
- 地图渲染需要3D视觉效果（或至少伪3D）

**影响：**
- 技术实现复杂
- 可能需要重写寻路算法
- 响应式布局更难做

**解决方案：**
```
方案A：简化高低差
- 不是真正的Z轴，而是"标签"
- 高地格子打标签"high"
- 判断时检查标签，不需要3D距离

方案B：延期实现
- MVP版本只做地形系统
- 高低差放在后续更新

方案C：视觉欺骗
- 用CSS阴影/颜色区分高地
- 逻辑上还是2D，只是有加成规则

推荐：方案C（平衡体验和开发成本）
```

### 问题 #5：动态战场变化的技术挑战

**描述：**
- Boss战地图会动态变化（添加毒沼、火坑等）
- 可能改变玩家和敌人的位置
- 需要重新计算路径、射程等

**影响：**
- ECS架构需要支持世界状态突变
- 可能有很多边界情况bug

**解决方案：**
```
方案A：预先放置
- 毒沼等一开始就存在，只是"激活/未激活"
- 阶段变化时激活，而不是新增

方案B：简化变化
- 只改变敌人行为，不改变地图
- 例如：Boss狂暴（伤害×2），而不是改变地形

方案C：状态驱动
- 地图格子有"状态"属性
- 阶段变化时批量修改格子状态
- ECS的System监听状态变化

推荐：方案A（最稳定）
```

## 🟡 中等问题（建议解决）

### 问题 #6：徽章更换机制不明确

**问题：**
- 更换徽章时"失去专属卡"
- 但卡组里已经有这些卡怎么办？

**解决方案：**
```
明确规则：
- 卡组中已有的专属卡保留（可以继续使用）
- 但无法再获得新的（从奖励中移除）
- 如果删除（从卡组移除），就无法再加回来

这样玩家的选择：
- 换徽章 = 旧专属卡用一张少一张
- 需要谨慎决定
```

---

### 问题 #7：战场掉落武器卡导致卡组膨胀

**问题：**
- 武器卡"永久持有"
- 但卡组上限25张
- 掉落多了会撑爆卡组

**解决方案：**
```
规则修改：
- 武器卡不占用卡组上限
- 单独 Weapon Deck（武器卡组）
- 每场战斗前可以选择带0-3张武器卡
- 或者：武器卡就是消耗品，有使用次数
```

---

### 问题 #8：经典模式解锁内容的可用性模糊

**问题：**
- 经典模式解锁隐藏徽章
- 但没说明普通模式能不能用

**解决方案：**
```
明确规则：
- 隐藏徽章所有模式通用
- 鼓励玩家挑战经典模式解锁
- 但不想挑战的玩家也能在其他模式用
```

---

### 问题 #9：地形效果过于复杂（9种地形）

**问题：**
- 需要记忆太多地形效果
- 15分钟游戏不适合

**解决方案：**
```
简化到5种：
1. 平地 - 无效果（默认）
2. 森林 - +20%防御（纯正面）
3. 高地 - 高低差优势（核心机制）
4. 陷阱 - 伤害（火坑/毒沼合并）
5. 障碍 - 阻挡移动（木箱/栅栏合并）

删除：河流（和陷阱重复）、冰面（随机性太强）、山地（和森林重复）
```

---

### 问题 #10：15分钟与系统复杂度矛盾

**问题：**
- 火焰纹章战棋+地形+高低差+连携+熟练度+徽章
- 系统太多，15分钟难以体验完整

**解决方案：**
```
MVP优先级：
P0（必须有）：徽章系统、卡牌系统、意图可视化
P1（重要）：地形系统（5种）、熟练度（简化版）
P2（加分）：连携系统、高低差
P3（后续）：动态战场变化、隐藏徽章

先保证核心循环爽，再逐步添加
```

### 问题 #12：经典模式难度过高

**问题：**
- 经典模式永久死亡
- 但解锁隐藏徽章需要通关第1/2/3层
- 对普通玩家太难

**解决方案：**
```
降低门槛：
- 经典模式通关第1层 → 解锁死灵法师
- 经典模式到达第2层（不需要通关Boss） → 解锁刺客
- 经典模式到达第3层 → 解锁风暴领主

这样玩家只需要"到达"而不是"通关"，难度降低
```

## ✅ 修复后的设计总结

### 修改清单

| 问题 | 修复方案 | 影响 |
|------|---------|------|
| 熟练度门槛高 | E→S只需20张 | 每局都能体验到成长 |
| 三角包围无法实现 | 删除，只保留相邻连携 | 简化系统 |
| 资源太复杂 | 移动消耗能量，删除移动力 | 只剩能量+徽章资源 |
| 高低差技术难 | 用标签模拟，视觉欺骗 | 保留体验，降低开发成本 |
| 动态战场难 | 改为激活/未激活 | 稳定实现 |
| 徽章换卡规则 | 已有卡保留但不能新增 | 明确策略选择 |
| 武器卡膨胀 | 单独武器卡组 | 不影响主卡组构筑 |
| 经典解锁可用性 | 全模式通用 | 激励玩家挑战 |
| 地形太复杂 | 5种简化 | 易于记忆 |
| 15分钟vs复杂度 | MVP分优先级 | 保证核心体验 |

### 修复后的核心循环

```
【简化版】
- 能量：用于移动（1格=1能量）和打卡牌
- 徽章：提供特殊机制（怒气/守护/元素/灵动）
- 地形：5种简单效果
- 熟练度：快速成长，每局都能到S级
- 连携：相邻即可（玩家+召唤物）
```
